Untracked file vs Tracked file

Accessing a Sketch file on Abstract from a branch other than your current branch results in an Untracked file being created on the platform. This happens because the file exists in the project you are working on, but it is not committed to your current branch. In contrast, a Sketch file that has been worked on and committed in your current branch is considered a Tracked file.

Accessing an Untracked File

On Abstract, accessing an untracked file only allows you to view the file. You can encounter an untracked file in the following situations:

  • Viewing a file on the main branch without creating a new branch.
  • Viewing a file on a contributor's branch without creating your own child branch.
  • Viewing a file from a previous commit on the main branch, a contributor's branch, or your own branch.
  • Viewing a file in your current branch without the latest changes.
  • Accessing a file on any branch, including the main branch, or from a previous commit, to copy and paste elements into a different file.

Viewing an Untracked File

  1. Open the Abstract desktop app.
  2. Select Project > Main.
  3. Select Files, choose the .sketch file you want to view, and then select Edit in Sketch > Open Untracked.

    4. Optionally, select Commits:

  4. Select a commit to view. In the file preview, choose the file and then select Open Untracked.
Note: Currently, Abstract does not support changes in an untracked file. If you need to make changes to a file, it is recommended to create a branch and then commit the changes for a merge.
